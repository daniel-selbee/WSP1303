Client: St. Francis Hospital for Animals

Discovery: Project Analysis

Site Intent

Cleaning up the code is one of the most important priorities of redesigning the site. The second is utilizing Gestalt principles in the site, especially in the text elements. The third priority will be the performance of the site. The site contains unnecessary elements that slow down its loading times even though the site is extremely simple. The site's graphic optimization needs to be significantly reworked to accomplish this. Next is implementing forms directly into the site so that clients do not have to print documents, fill them out, and turn them into the veterinary office. From experience, I know that veterinary clinics must keep records for at least 7 years. This would be so much easier if this data were stored on the computer. The last category is responsiveness. Many people use their mobile devices when trying to find a business. Since this is a hospital, it is even more important that people are able to find the business quickly from their mobile devices.

Code

Reworking the code is not going to be a simple task. Much of the site contains unnecessary java elements that can be replaced by CSS. Replacing these elements will resolve some of the performance issues, but not all of them. The site has 5 errors while validating that I plan to fix. I expected many more when I entered it into the validation site. The code is also older. HTML 5 and CSS3 would greatly benefit this site. The final aspect to correct is the spacing and indentation issues. The code is very hard to read through efficiently.

Gestalt Principles

The five Gestalt principles that I am going to focus on are similarity, proximity, continuity, and closure. I need to change most of these elements in this site. I'll start with similarity. As I clicked through the separate links, a red flag went off. The pages looked different! I would fix this with simple design elements that tie together with proximity. Each page would contain this style of aesthetics. The other thing that is lacking is continuity. The eye jumps all over the page making it confusing for the user at first. I will fix this with a top-left approach. The heading will stick out, drawing the eye there. For closure, I am going to create boxes for different subjects that don't quite touch, but have a good proximity.

Performance and Image Optimization

I think much of the performance issues come from the embedded javascript. The slideshows are unnecessary and are much too small when compared to other elements. I think this could be replaced by a gallery page or even a CSS slideshow that is clickable and doesn't play automatically. The images in the site need to be optimized for responsiveness. For example, I will retrace the logo with the pen tool in AI and make it a PNG so it can scale for larger monitors. It looks very pixelated on my 27" iMac.

Integrated Forms

The current site tries to encourage its prospective clients to print off and bring in forms to the veterinary clinic for processing. I will improve this process by building HTML forms that the clients can fill out online so they can be stored in the computer. This will make it easier on the clients and the veterinary clinic to keep track of the forms. This registration process also allows the receptionist to look up the new clients instantly and schedule appointments without having to input forms.

Responsiveness

Responsiveness is a very important aspect of this site. As I stated in the introductory paragraph, it is vital that clients are able to access the site from their mobile devices to find the veterinary hospital in an emergency. It is also a great way for people to just find a new veterinary clinic if they aren't happy with their current vet, or if they currently don't have a vet.
